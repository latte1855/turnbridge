# ✅ **E0504 — 中獎清冊資料檔（Award）**

---

# 📌 1. 訊息用途（MIG 原文）

> 「中獎清冊資料檔」提供營業人查詢其買受人載具發票是否中獎。
> 由平台定期產生，買受人資訊依隱私規範遮蔽。

---

# 📌 2. 資料結構（E0504 為「平面結構」而非樹狀）

E0504 **不像** E0501/E0502/E0503 有 Main/Details，
它是一個 **單一 Record 一筆一筆羅列** 的檔案。

以下為 MIG 結構：

```
Award
 ├── InvoiceNumber
 ├── InvoiceDate
 ├── AwardType
 ├── AwardAmount
 ├── CarrierType
 ├── CarrierId1
 ├── CarrierId2?
 ├── PrizeLevel
 └── MaskedBuyerId?
```

---

# 📌 3. 欄位表（依 MIG 格式）

⚠ 由於 MIG4.1 的中獎清冊資料欄位被定義在「字軌/進項附錄段落」內，
欄位內容已依官方教材（財政資訊中心）交叉比對整理如下。

| 欄位                | 必要性 | 說明               |
| ----------------- | --- | ---------------- |
| **InvoiceNumber** | M   | 中獎發票號碼（字軌 + 8 碼） |
| **InvoiceDate**   | M   | 開立日期（YYYYMMDD）   |
| **AwardType**     | M   | 中獎獎項類別（雲端三獎等）    |
| **AwardAmount**   | M   | 中獎金額             |
| **CarrierType**   | M   | 載具種類（例如 3J0002）  |
| **CarrierId1**    | M   | 載具號碼             |
| **CarrierId2**    | O   | 第二載具號碼           |
| **PrizeLevel**    | M   | 中獎等級（1~n）        |
| **MaskedBuyerId** | O   | 買受人識別碼（遮蔽處理後）    |

**特別注意：**

* 若載具為手機條碼，CarrierType = **/XX** 格式（如 3J0002）。
* MaskedBuyerId 通常只有境外電商等 B2C 特殊案例會出現。

---

# 📌 4. E0504 與一般 Invoice 差異

| 項目   | Invoice（F0401/E0502）    | E0504（中獎清冊） |
| ---- | ----------------------- | ----------- |
| 資料來源 | 賣方 / 平台                 | 平台（財政部）     |
| 內容   | 完整發票資料                  | 中獎資訊（有限欄位）  |
| 主題   | 課稅 / 核銷                 | 統一發票對獎      |
| 結構   | Main + Details + Amount | 單層欄位        |

E0504 用途極單純：
**營業人查詢顧客是否中獎（不含個資）**。

# 📌 5. XML 範例（官方教材格式）

```

    <InvoiceDate>20231108</InvoiceDate>

</Award>
```

## 1. Turnkey 是什麼？

Turnkey（電子發票傳輸工具）是財政部提供給營業人使用的上傳程式。
企業可利用 Turnkey 將自行產生的電子發票 XML 或其他格式資料：

- 打包
- 簽章
- 上傳至電子發票整合服務平台（MOF）
- 下載平台回覆結果
- 完成整體電子發票生命週期處理

Turnkey 為 **B2B 發票交換**、**存證發票（B2C/B2B）上傳** 的必備工具。

---

## 2. Turnkey 角色與企業端分工

Turnkey 主要負責：

1. 驗證資料格式（MIG）  
2. XML 打包與簽章  
3. 上傳至 MOF  
4. 下載 ProcessResult  
5. 回寫內部資料庫（STATUS、錯誤碼等）

企業端主要負責：

1. 發票資料的產生（開立）  
2. MIG 欄位檢核（必要）  
3. 字軌管理、生命週期控管  
4. 錯誤處理與補傳  
5. 系統對接（ERP / POS / OMS）

---

## 3. Turnkey 版本與適用範圍

Turnkey v3.9 支援以下標準：

- **MIG v3.x**：B2B、B2C 舊制存證、折讓
- **MIG v4.x**：F/G 系列（平台存證）
- **B2SSTORAGE**：MIG 4.0+ 新架構

自 MIG 4.0 起：

- A/B/C/D 系列整併為 F/G 系列
- Turnkey Upload/Download 需切換到 B2S 目錄

---

## 4. Turnkey 的重要性

電子發票採用 XML，並需通過：

- **欄位檢核**（MIG 格式）
- **規則檢核**（金額試算、稅額校驗）
- **生命週期檢核**（開立 → 作廢 → 註銷）
- **簽章驗證**
- **上傳傳輸控制**
- **平台回應處理**

Turnkey 將企業端與政府平台串起，減少錯誤、確保上傳成功。
